# Note-Taking App — Backend

Go + Gin REST API with PostgreSQL storage and JWT authentication.

## Prerequisites

- **Go** 1.26+
- **PostgreSQL** running locally (14+ recommended)
- `openssl` (used by `make env` to auto-generate `JWT_SECRET`; present by default on virtually all Linux/macOS installs)
- `sudo` access to your local Postgres install (used by `make db-create`/`db-drop`, matching a default Arch/systemd-style local install — see Troubleshooting if your setup differs)

All commands below are run from this directory (`backend/`).

## Quick start

```bash
make setup   # env file (secret auto-generated) + tools + deps + database + migrations
make run     # starts the API on :8080
```

No manual editing required — `make setup` creates `.env` from `.env-template` (with a random `JWT_SECRET` filled in) and runs the rest of the setup against it in one shot. It's idempotent — safe to re-run; it won't overwrite an existing `.env` or regenerate an existing secret.

The `.env-template` defaults (`DB_HOST=localhost`, `DB_USER=postgres`, empty `DB_PASS`, `DB_NAME=notes_app`) assume a local Postgres reachable via peer/trust auth, matching a fresh Arch/systemd-style install. If your Postgres setup differs, edit `.env` before running `make setup` (or re-run `make db-create migrate-up` after editing).

| Key           | Meaning                                                                      | Example                      |
| ------------- | ---------------------------------------------------------------------------- | ---------------------------- |
| `DB_HOST`     | Postgres host                                                                | `localhost`                  |
| `DB_PORT`     | Postgres port                                                                | `5432`                       |
| `DB_USER`     | Database role                                                                | `postgres`                   |
| `DB_PASS`     | Role password                                                                | —                            |
| `DB_NAME`     | Database name                                                                | `notes_app`                  |
| `SERVER_PORT` | HTTP port for the API                                                        | `8080`                       |
| `APP_ENV`     | `development` (default) or `production` — gates whether Swagger UI is served | `development`                |
| `JWT_SECRET`  | Token signing secret (keep private)                                          | auto-generated by `make env` |

`.env` is gitignored — never commit it.

## Running the server

```bash
make run     # go run ./cmd/server — no build artifact
# or
make build   # compiles to bin/server
./bin/server
```

The API listens on `http://localhost:8080`.

> The listen port is currently hardcoded to `:8080` in `cmd/server/main.go` — changing `SERVER_PORT` in `.env` has no effect yet.

Once it's up, interactive API docs are at `http://localhost:8080/swagger/index.html` (dev only, disabled when `APP_ENV=production` — see [`docs/SWAGGER.md`](../docs/SWAGGER.md)).

## Troubleshooting

**`make db-create` / `make db-drop` fail or hang on a password prompt.**
These use `sudo -u postgres psql`, matching a default local Postgres install (peer auth for the `postgres` OS/DB role). If your Postgres is configured differently (Docker, remote host, a non-default role), skip these targets, create/drop the database yourself with whatever method matches your setup, then continue with `make migrate-up`.

**Postgres isn't running.**

```bash
sudo systemctl start postgresql
```

First-time installs may need `sudo -u postgres initdb -D /var/lib/postgres/data` before the service will start.

**`make migrate-up` / `make migrate-down` can't connect.**
These build a connection string directly from `DB_HOST`/`DB_PORT`/`DB_USER`/`DB_PASS`/`DB_NAME` in `.env` (`sslmode=disable`). Confirm those values are correct and that Postgres accepts TCP connections on `DB_HOST:DB_PORT` for `DB_USER` (check `pg_hba.conf` if you get an auth error rather than a connection-refused error).

**`swag`/`migrate` not found even after `make tools`.**
`make tools` installs them to `$(go env GOPATH)/bin`. The Makefile calls them by full path, so `make swagger`/`make migrate-up` work regardless — but if you want to invoke them directly, add that directory to your `PATH`.

## API overview

Public endpoints:

| Method | Path                  | Purpose                                       |
| ------ | --------------------- | --------------------------------------------- |
| POST   | `/auth/signup`        | Create an account                             |
| POST   | `/auth/login`         | Log in, returns JWT tokens                    |
| POST   | `/auth/refresh-token` | Exchange refresh token for a new access token |

Protected endpoints (require `Authorization: Bearer <access token>`):

| Method | Path               | Purpose                                     |
| ------ | ------------------ | ------------------------------------------- |
| POST   | `/api/notes`       | Create a note                               |
| GET    | `/api/notes/:id`   | Get a note                                  |
| PUT    | `/api/notes`       | Update multiple notes in one batch          |
| DELETE | `/api/notes`       | Delete multiple notes in one batch          |
| GET    | `/api/folders`     | List folders                                |
| POST   | `/api/folders`     | Create a folder                             |
| GET    | `/api/folders/:id` | Get a folder (with its child folders/notes) |
| PUT    | `/api/folders`     | Update multiple folders in one batch        |
| DELETE | `/api/folders`     | Delete multiple folders in one batch        |

Full request/response shapes are documented via Swagger — see [`docs/SWAGGER.md`](../docs/SWAGGER.md) for how to view and regenerate them.
